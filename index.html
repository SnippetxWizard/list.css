1. Code in Dark

Rules for Code in Dark:

  • Participants must write code in a dark-themed editor.
  • No external plugins or themes are allowed.
  • Code must be written in a single session without breaks.
  • Submissions will be judged on functionality and creativity.
2. Debugging

Rules for Debugging:

  • Participants will be given a buggy code snippet.
  • They must identify and fix all bugs within the given time.
  • No external tools or debuggers are allowed.
  • Submissions will be judged on accuracy and speed.
3. UI/UX

Rules for UI/UX:

  • Participants must design a user-friendly interface for a given problem statement.
  • Tools like Figma, Adobe XD, or Sketch can be used.
  • Submissions must include wireframes and a brief explanation of the design.
  • Judging criteria: usability, creativity, and adherence to the problem statement.
